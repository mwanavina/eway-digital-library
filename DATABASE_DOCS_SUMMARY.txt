================================================================================
TABLES CREATED
================================================================================

1. schools
   - Top-level organizational units
   - Columns: id, name (UNIQUE), created_at
   - Relationships: 1:N with departments

2. departments
   - Academic departments within schools
   - Columns: id, name (UNIQUE), school_id (FK), created_at
   - Relationships: N:1 with schools, 1:N with programs

3. programs
   - Academic programs within departments
   - Columns: id, name (UNIQUE), department_id (FK), created_at
   - Relationships: N:1 with departments, 1:N with courses

4. courses
   - Individual courses within programs
   - Columns: id, name, code (UNIQUE), program_id (FK), created_at
   - Relationships: N:1 with programs, 1:N with documents

5. levels
   - Academic levels (Year 1-5)
   - Columns: id, level_number (UNIQUE), description, created_at
   - Relationships: 1:N with documents (optional)

6. documents
   - Educational resources (papers, journals, dissertations, etc.)
   - Columns: 17 columns including file metadata and status
   - Relationships: N:1 with courses, N:1 with levels, 1:N with download_logs
   - Fields: title, year, semester, exam_type, file info, upload status, etc.

7. users
   - System users (admin and regular users)
   - Columns: id, username (UNIQUE), email (UNIQUE), role, created_at
   - Roles: admin | user
   - Relationships: 1:N with sessions, 1:N with download_logs

8. sessions
   - User session management
   - Columns: id, user_id (FK), token (UNIQUE), created_at, expires_at
   - Relationships: N:1 with users

9. download_logs
   - Download activity tracking
   - Columns: id, document_id (FK), user_id (FK), downloaded_at
   - Relationships: N:1 with documents, N:1 with users

DATA TYPES
================================================================================

INTEGER
  - Used for: IDs, counts, years, semesters, file sizes
  - Examples: schools.id, documents.year, documents.download_count

CHARACTER VARYING (VARCHAR)
  - Used for: Names, titles, codes, email, username, paths, URLs
  - Max lengths: 50-500 characters depending on field
  - Examples: schools.name, documents.title, courses.code

TEXT
  - Used for: Long-form content
  - Examples: documents.error_message

TIMESTAMP (WITHOUT TIME ZONE)
  - Used for: Date/time tracking
  - Always assumed to be UTC
  - Examples: created_at, uploaded_at, downloaded_at, expires_at

SERIAL (AUTO-INCREMENT)
  - Used for: Primary key IDs
  - Auto-incrementing integer
  - Examples: All id columns

CONSTRAINTS REFERENCE
================================================================================

PRIMARY KEYS (9):
  - schools.id, departments.id, programs.id, courses.id, levels.id,
    documents.id, users.id, sessions.id, download_logs.id

UNIQUE CONSTRAINTS (8):
  - schools.name
  - departments.name
  - programs.name
  - courses.code
  - levels.level_number
  - users.username
  - users.email
  - sessions.token

FOREIGN KEY CONSTRAINTS (8):
  - departments.school_id → schools.id
  - programs.department_id → departments.id
  - courses.program_id → programs.id
  - documents.course_id → courses.id
  - documents.level_id → levels.id
  - download_logs.document_id → documents.id
  - download_logs.user_id → users.id
  - sessions.user_id → users.id
  All with ON DELETE CASCADE, ON UPDATE CASCADE

CHECK CONSTRAINTS (4):
  - documents.semester IN (1, 2) OR NULL
  - documents.upload_status IN ('pending', 'completed', 'failed')
  - users.role IN ('admin', 'user')
  - levels.level_number > 0 AND level_number <= 10
